# CI/CD

Put the project on GitHub and add automated checks.

Do this after the app runs locally and in local Kubernetes. CI/CD is most useful when it checks a project you already know works.

## Phases

| Phase | Open | Finish with |
| --- | --- | --- |
| 15 | [Git And GitHub](01-git-github.md) | Local repo connected to GitHub |
| 16 | [GitHub Actions CI](02-github-actions-ci.md) | CI workflow running on every push |

## Section Checkpoint

Before moving to AWS:

- `git status` is clean.
- `git push` works.
- GitHub Actions starts automatically.
- The workflow passes or you understand exactly why it failed.
